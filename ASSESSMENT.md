# ASSESSMENT — Agent-Native 成熟度自评模板

> 派生自 `SPEC.md` 四公理与 `CHECKLIST.md` 条目（单一真源在 SPEC，改 SPEC 须同步本表 —— 公理一）。
> 用途：任意项目复制本表填写 → **机械**输出 L0–L3 评级 + 缺口清单。本表自身即可当评分脚本读（见 §5）。
> 判级硬条件**仅看 MUST**（与 SPEC §3.2「满足公理X = 该公理全部 MUST 满足」一致）；
> `[SHOULD]` 条目不影响级别，仅汇入「推荐改进」。关键词依 RFC 2119。

- [ ] `A4.4` [SHOULD] 最小权限 + 变更可审计

## 2. 确定性判级算法

定义 `MUST(k)` = 公理 k 的 MUST 条目集合：
`MUST(1)={A1.1,A1.2,A1.3}`、`MUST(2)={A2.1,A2.2}`、`MUST(3)={A3.1,A3.2,A3.3}`、`MUST(4)={A4.1,A4.2}`。
`sat(k)` = 公理 k 的**全部** MUST 均勾选 `[x]`（布尔）。

```
输入: 每条勾选位 (true/false)
sat[k] = AND(MUST(k) 全部为 true)        # k = 1..4

# 级别 = 从公理一起，连续满足公理的最长前缀
if not sat[1]:            level = "L0"   # 公理一未满 → 人类原生
elif not sat[2] or not sat[3]:           # 公理二、三任一未满，止于 L1
                         level = "L1"
elif not sat[4]:         level = "L2"
else:                    level = "L3"

# 报告：始终列出 sat[1..4] 四位真值，使「部分满足」可读
#   例 sat=[T,F,T,F] → level=L1，并标注「公理三已达标但因公理二缺口被前缀规则挡在 L1」
```

判级要点（明确规则，便于脚本与人对齐）：

- **前缀规则**：级别取「公理一起连续 `sat` 为真的前缀长度」对应的级别。公理有先后依赖（L2 要求 ①②③，L3 再要 ④），故**跳级满足不升级**——即便 `sat[4]=true`，只要 `sat[2]` 或 `sat[3]` 为假，仍停在 L1。
- **公理二、三同档**：二者共同构成 L1→L2 的门槛，缺任一即 L2 不达标（与 SPEC §3.2「L2 = +公理二、三」一致）。
- **SHOULD 不参与判级**：`[SHOULD]` 条目缺失不降级，仅进入 §3 的「推荐改进」。满足全部 MUST 且 SHOULD 有缺 → 级别照给，附 `SHOULD-gap` 标记。
- **四问一致性校验**（可选硬门）：若 `Qk == yes` 但 `sat[k] == false`（或反之），输出 `INCONSISTENT(k)`，判级以勾选位为准、四问答案须复核。

## 3. 缺口清单生成规则

```
# 距下一级的缺口 = 下一级所需、当前未勾的 MUST
target = next_level(level)                # L0→L1→L2→L3；L3 无 next
gap_to_next = { 条目 ∈ MUST(达成 target 所需公理) : 勾选位 == false }
#   L0→L1 需补 MUST(1) 中未勾者
#   L1→L2 需补 MUST(2)∪MUST(3) 中未勾者
#   L2→L3 需补 MUST(4) 中未勾者
should_gap  = { 所有 [SHOULD] 条目 : 勾选位 == false }   # 推荐改进，不阻级别
```

输出格式（机械生成，逐条引用条目 ID + CHECKLIST 文案）：

```
评级: L{n}
sat: [①=_, ②=_, ③=_, ④=_]
距下一级 (L{n}→L{n+1}) 还差:
  - <ID> <条目文案>        # 逐条列未勾 MUST，按上方 target 集合
推荐改进 (SHOULD, 不影响级别):
  - <ID> <条目文案>
四问一致性: OK | INCONSISTENT(k=...)
```

- 已达 L3 时 `距下一级` 为空，若 SHOULD 全勾则标 `已满分 (含 SHOULD)`。
- 缺口**只列阻挡下一级的 MUST**，不列更高级别的（避免噪声）；要看全貌看 `sat` 四位即可。

## 4. 填好的示例

假想项目 **`acme-deploy`（一个部署 CLI）**，评估人填写如下。

四问：Q1=yes　Q2=no　Q3=yes　Q4=no

逐条勾选：

| 公理一 | 公理二 | 公理三 | 公理四 |
| --- | --- | --- | --- |
| A1.1 `[x]` | A2.1 `[x]` | A3.1 `[x]` | A4.1 `[ ]` |
| A1.2 `[x]` | A2.2 `[ ]` | A3.2 `[x]` | A4.2 `[ ]` |
| A1.3 `[x]` | A2.3 `[ ]` | A3.3 `[x]` | A4.3 `[x]` |
| A1.4 `[x]` | A2.4 `[ ]` | A3.4 `[x]` | A4.4 `[ ]` |
| A1.5 `[x]` | A2.5 `[ ]` |  |  |
| A1.6 `[ ]` |  |  |  |

机械求值：
`sat[1]` = A1.1∧A1.2∧A1.3 = **true**（A1.6 是 SHOULD，不计）
`sat[2]` = A2.1∧A2.2 = true∧false = **false**
`sat[3]` = A3.1∧A3.2∧A3.3 = **true**
`sat[4]` = A4.1∧A4.2 = false∧false = **false**

前缀规则：sat[1]=T → 过 L0；sat[2]=F → 止步。**级别 = L1**。

输出：

```
评级: L1（契约就绪）
sat: [①=T, ②=F, ③=T, ④=F]
距下一级 (L1→L2) 还差:        # 需补 MUST(2)∪MUST(3) 中未勾者
  - A2.2 失败机器可分类（错误码/类型），非仅自然语言
                              # 注：MUST(3) 已全勾，③本已达标，仅因②缺口被前缀规则挡在 L1
推荐改进 (SHOULD, 不影响级别):
  - A1.6 约束规则分层：强制配置 与 参考叙事 分开
  - A2.3 失败分类含可操作维度（错误域 + 是否可重试）
  - A2.4 产物带 provenance
  - A2.5 构建/产出可复现
  - A4.3 / A4.4 见公理四（该公理 MUST 亦未满，距 L3 尚远）
四问一致性: OK   # Q1=yes↔sat①=T, Q2=no↔sat②=F, Q3=yes↔sat③=T, Q4=no↔sat④=F 全一致
```

解读：`acme-deploy` 契约层达标（L1），但失败只 print 散文（A2.2 未过）使②不达标，于是即便③已全绿、也只能停在 L1——这正是前缀规则「跳级不升级」。补上 A2.2 一条即可升 L2。安全公理两条 MUST 全缺，距 L3 还远。

## 5. 本模板如何对齐公理①（确定性契约）

- **机械可判定**：级别仅是勾选位的纯布尔函数（§2），无人读裁量；同一份勾选输入恒定得同一级别与同一缺口集——可复现。
- **可当评分脚本读**：条目有稳定 ID（`A{公理}.{序}`）+ `[MUST]/[SHOULD]` 机器可读标注 + `[x]/[ ]` 勾选位；解析 markdown 勾选框 → 套 §2 规则 → 套 §3 规则，即得评级与缺口，全程不解析自然语言。
- **单一真源 + 派生**：判定语义的真源是 `SPEC.md` 的 MUST/SHOULD；本表与 `CHECKLIST.md` 共用条目 ID，文案引用而非另立标准。改 SPEC → 改 CHECKLIST → 核对本表 ID 集合，三者一致（公理①的「派生物锁一致」）。
- **强制配置 vs 参考叙事分层**：参与判级的是 `[MUST]` 标注与勾选位（强制配置）；§4 解读、本节说明是参考叙事，删去不影响判级——契约与叙事分离。
