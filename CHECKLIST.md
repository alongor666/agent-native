# CHECKLIST — Agent-Native 合规验收

派生自 `SPEC.md` 的四公理 MUST（改 SPEC 须同步此清单 —— 单一真源，公理一）。
用法：对目标项目逐条勾选；任一 MUST 未满 = 该公理不达标。
每条附「→ 验证」一句，描述**如何机械判定**（技术无关，可据此写评分脚本）。

## 四问速判（机械可执行）

- [ ] **契约**：陌生 Agent 仅靠机器可读文件（不读散文、不问人），能否正确调用/消费/扩展？
  → 验证：把所有 `.md`/散文从输入中剥离，仅留机器可读文件喂给一个无先验 Agent，令其完成一次真实调用/消费；成功即过。
- [ ] **反馈**：执行任一操作后，能否程序化判断成功/失败/失败原因？
  → 验证：脚本读返回值/退出码/结构化输出即可分支处理，全程不解析自然语言。
- [ ] **恢复**：任一长流程中断后重跑，是否安全且不重复副作用？
  → 验证：在中途 kill，再次运行同一命令；终态与副作用计数（写入/发送次数）须与一次跑通一致。
- [ ] **安全**：能否在不造成不可逆损害的前提下自由试错？
  → 验证：枚举所有不可逆/外发/破坏性操作，逐一确认其前置有确认门或可回滚；有一个裸奔即不过。

## 公理一 · 确定性契约

- [ ] 接口契约机器可读（类型/Schema/IDL），非仅散文
  → 验证：存在可被解析器加载的契约文件（schema/类型/IDL 解析通过、非空），且覆盖全部公开入口。
- [ ] 同一事实单一真源 + 派生物自动生成 + 自动校验锁一致
  → 验证：重跑生成命令后 `diff` 派生物为空；篡改派生物后校验门禁须报错（漂移被拦）。
- [ ] 版本兼容性机器可判定（SemVer + 显式兼容语义）
  → 验证：版本号符合 SemVer 正则；存在机器可读的兼容性规则/changelog，可程序判定两版本是否破坏。
- [ ] 能力/入口有机器可读发现清单
  → 验证：存在固定路径的发现清单（如 manifest/llms.txt/AGENTS.md），解析后列出的入口与实际一一对应。
- [ ] 人读叙事 与 Agent 契约 分离
  → 验证：契约文件与叙事文档不同文件；删掉所有叙事文档，契约仍完整可用。
- [ ] 约束 Agent 的规则分层：机械可执行/拒止的强制配置 与 参考叙事 分开
  → 验证：存在被工具实际读取并能拒止违规操作的配置（如 deny/allow 规则）；纯散文规则不计入硬约束。

## 公理二 · 可验证闭环

- [ ] 每操作返回机器可读结果（结构化输出 + 语义化退出码/状态）
  → 验证：每操作的成功路径返回可解析结构（JSON/对象/枚举）且退出码/状态与语义一致，无「静默成功」。
- [ ] 失败机器可分类（错误码/类型），非仅自然语言消息
  → 验证：每类失败带稳定 code/类型，可被 `==`/模式匹配分支；改文案不影响分类。
- [ ] 失败分类含可操作维度（错误域 + 是否可重试）
  → 验证：失败结果可程序化读出错误域与可重试标记，据此自动选择重试/改参/放弃，无需人读。
- [ ] 产物带 provenance（输入指纹 + 代码版本 + 执行者 + 时间）
  → 验证：产物旁存在元数据，四要素字段非空且可解析；据此能另起一次独立复算并比对。
- [ ] 构建/产出可复现
  → 验证：同输入两次构建，产物哈希一致（或差异仅在白名单的时间戳字段内）。

## 公理三 · 可恢复状态

- [ ] 长流程幂等或可断点续跑，中断重跑不重复副作用
  → 验证：中途 kill 后重跑，副作用计数不增、数据无重复/损坏（同「恢复」四问）。
- [ ] 状态外置于持久载体，非仅存于上下文/隐藏全局态
  → 验证：进程结束后状态仍可从持久载体（文件/库/对象存储）读出；冷启动可据此续跑。
- [ ] 运行期约束/规则也外置于持久载体，非仅存于会话对话
  → 验证：清空/压缩会话上下文后，约束仍生效（来自持久配置而非对话记忆）。
- [ ] 操作幂等（幂等键/期望状态收敛）
  → 验证：同一请求带同一幂等键执行 N 次，终态等同执行 1 次。

## 公理四 · 安全边界

- [ ] 不可逆/外发/破坏性操作 显式确认或可回滚
  → 验证：默认（无确认标志）调用该类操作时被拒绝或仅预览；或执行后存在可程序触发的回滚路径。
- [ ] 硬性安全边界由外部机制强制（沙箱/权限/拒止规则），AI 自检非唯一防线
  → 验证：禁用/绕过 Agent 自身判断后，边界仍由外部层（OS 沙箱/权限系统）拦截不可逆操作。
- [ ] 提供 dry-run/preview
  → 验证：存在 dry-run 开关，开启后无任何真实副作用、仅输出将发生的变更。
- [ ] 最小权限 + 变更可审计
  → 验证：授予的权限集可枚举且无多余项；每次变更留下不可否认的审计记录（谁/何时/改了什么）。
