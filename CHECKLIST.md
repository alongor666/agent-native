# CHECKLIST — Agent-Native 合规验收

派生自 `SPEC.md` 的四公理 MUST（改 SPEC 须同步此清单 —— 单一真源，公理一）。
用法：对目标项目逐条勾选；任一 MUST 未满 = 该公理不达标。

## 四问速判（机械可执行）

- [ ] **契约**：陌生 Agent 仅靠机器可读文件（不读散文、不问人），能否正确调用/消费/扩展？
- [ ] **反馈**：执行任一操作后，能否程序化判断成功/失败/失败原因？
- [ ] **恢复**：任一长流程中断后重跑，是否安全且不重复副作用？
- [ ] **安全**：能否在不造成不可逆损害的前提下自由试错？

## 公理一 · 确定性契约

- [ ] 接口契约机器可读（类型/Schema/IDL），非仅散文
- [ ] 同一事实单一真源 + 派生物自动生成 + 自动校验锁一致
- [ ] 版本兼容性机器可判定（SemVer + 显式兼容语义）
- [ ] 能力/入口有机器可读发现清单
- [ ] 人读叙事 与 Agent 契约 分离

## 公理二 · 可验证闭环

- [ ] 每操作返回机器可读结果（结构化输出 + 语义化退出码/状态）
- [ ] 失败机器可分类（错误码/类型），非仅自然语言消息
- [ ] 产物带 provenance（输入指纹 + 代码版本 + 执行者 + 时间）
- [ ] 构建/产出可复现

## 公理三 · 可恢复状态

- [ ] 长流程幂等或可断点续跑，中断重跑不重复副作用
- [ ] 状态外置于持久载体，非仅存于上下文/隐藏全局态
- [ ] 操作幂等（幂等键/期望状态收敛）

## 公理四 · 安全边界

- [ ] 不可逆/外发/破坏性操作 显式确认或可回滚
- [ ] 提供 dry-run/preview
- [ ] 最小权限 + 变更可审计
